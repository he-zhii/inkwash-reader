```

[公众号二维码图片]
[公众号名/关注数/文章数等统计]

```

### 需要清洗的内容类型

| 内容类型 | 示例 | 处理策略 |
|----------|------|----------|
| 头像图片墙 | `![头像](https://wx.qlogo.cn/mmopen/...)` | **移除** — 纯干扰 |
| 公众号二维码 | `![](https://mmbiz.qpic.cn/...)` | **移除** |
| 点赞/转发提示 | "点赞，在看与转发是更好的支持！" | **移除** |
| 公众号卡片信息 | 公众号名、关注数、文章数 | **移除** |
| 付费/喜欢统计 | "1万+ 人付费，30 人喜欢" | **保留** — 作为文章元数据展示 |
| 评论区 | "暂无评论" 或实际评论 | **保留** — 有内容的评论保留 |
| 文内互链 | `[标题](http://mp.weixin.qq.com/s?...)` | **分类处理**（见下方） |
| 尾部推荐列表 | 文末大量链接堆叠 | **提取到独立区域** |

### 链接分析

| 链接类型 | 数量 | 说明 |
|----------|------|------|
| 微信公众号文章链接 | **1835 条** | 占绝大多数，分布在正文互引和尾部推荐 |
| 其他外部链接 | 6 条 | 极少 |

> [!IMPORTANT]
> 1835 条微信链接里有大量**重复**（每篇文章尾部都有一个近乎相同的推荐列表），去重后预计会大幅减少。这些链接分为两类：
> - **已收录**：指向你已经拥有的 21 篇文章之一
> - **未收录**：指向你尚未购买/保存的文章 → 这就是你说的"挖掘尚未收录内容"的价值所在

## 二、产品设计方向

### 核心定位

> 一个为个人使用的、针对中文长文阅读优化的**私人文章库**，兼具内容管理和发现功能。

### 页面架构

```
┌─────────────────────────────────────────────┐
│                  首页 / 文章列表              │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐       │
│  │ 文章卡片 │ │ 文章卡片 │ │ 文章卡片 │       │
│  │ 标题     │ │ 标题     │ │ 标题     │       │
│  │ 来源/日期│ │ 来源/日期│ │ 来源/日期│       │
│  │ 摘要     │ │ 摘要     │ │ 摘要     │       │
│  │ 标签     │ │ 标签     │ │ 标签     │       │
│  └─────────┘ └─────────┘ └─────────┘       │
│                                             │
│  [筛选：全部 | 已付费 | 免费 | 按公众号]      │
│  [排序：按日期 | 按字数 | 按阅读量]           │

│  │ 标题 / 作者 / 来源公众号 / 日期       │    │

│  状态管理：待购买 / 已忽略                    │
```

### 中文排版设计规范

针对中文长文阅读的专业排版：

| 排版项 | 建议值 | 说明 |
|--------|--------|------|
| 正文字体 | `"Noto Serif SC", "Source Han Serif SC", serif` | 宋体系衬线字体，适合长文阅读 |
| 标题字体 | `"Noto Sans SC", "Source Han Sans SC", sans-serif` | 黑体无衬线，层次分明 |
| 正文字号 | 17px | 屏幕阅读舒适尺寸 |
| 行高 | 1.85 | 中文最佳阅读间距 |
| 段间距 | 1.2em | 段落间明确留白 |
| 内容最大宽度 | 720px | 每行 35~40 中文字符 |
| 文字颜色 | `#2c2c2c`（浅色模式）/ `#d4d4d4`（深色模式） | 降低对比度，减轻视觉疲劳 |
| 首行缩进 | 不缩进 | 现代中文排版趋势 |
| 加粗文字 | 重点着色 + 加粗 | 原文中 `**文字**` 的强调内容用颜色区分 |

### 视觉风格方向

> [!TIP]
> 以下提供两种风格方向供选择，也可以混搭：

**方案 A — 墨韵书斋（中式文人风）**
- 色调：纸白底色 `#FAF8F5`，墨色文字，朱砂色点缀 `#B5544A`
- 风格：素雅留白多，接近纸质书阅读感
- 适合：沉浸式深度阅读
- 暗色模式：`#1a1a1e` 底色，温暖白文字

**方案 B — 现代阅读器（类 Notion/语雀风）**
- 色调：纯白底 `#FFFFFF`，深灰文字，蓝色调点缀 `#3B82F6`
- 风格：简约干净，工具感强，交互感好
- 适合：管理+阅读并重
- 暗色模式：`#0F0F0F` 底色，高对比

## 三、技术架构讨论

### 方案对比

| 方案 | 技术栈 | 优点 | 缺点 |
|------|--------|------|------|
| **A. 纯静态站** | HTML + CSS + JS | 极简、无需后端、可直接打开 | 新增文章需手动编辑代码 |
| **B. Vite SPA** | Vite + Vanilla JS/Vue | 热更新开发体验好、组件化 | 需 node 环境 |
| **C. Next.js SSG** | Next.js + MDX | 成熟的 Markdown 渲染生态 | 架构偏重 |

> [!IMPORTANT]
> **建议方案 A（纯静态站）**，原因：
> 1. 个人使用，21 篇文章体量有限
> 2. 无需服务器部署，双击 `index.html` 即可使用
> 3. 开发简单直接，后续维护成本低
> 4. 数据处理（清洗、链接管理）通过构建脚本一次性完成，生成干净的 JSON 数据

### 项目结构设想

```
pages_beautify/
├── raw/                          # 原始 Markdown 文件（保留不动）
│   ├── 五个影响你一辈子的思维误区.md
│   └── ...
├── scripts/
│   └── build.js                  # 内容清洗 & 构建脚本
├── dist/                         # 构建输出
│   ├── index.html                # 首页（文章列表）
│   ├── articles/
│   │   ├── article-1.html        # 各文章阅读页
│   │   └── ...
│   ├── links.html                # 待收录链接库页面
│   ├── css/
│   │   └── style.css             # 全局样式
│   ├── js/
│   │   └── app.js                # 交互逻辑
│   └── data/
│       ├── articles.json         # 清洗后的文章元数据
│       └── external-links.json   # 去重后的外部链接库
└── README.md
```

### 构建脚本的核心任务

```
原始 .md 文件
    ↓ 解析
提取元信息（标题/作者/来源/日期/阅读量/付费状态）
    ↓ 清洗
移除头像墙、二维码、公众号卡片、点赞提示等
    ↓ 链接处理
    ├── 文内链接：标记已收录/未收录
    ├── 尾部推荐列表：提取 → 去重 → 归入链接库
    └── 已收录链接：转为站内跳转
    ↓ 渲染
Markdown → HTML（正文、评论分离）
    ↓ 输出
生成各页面 HTML + JSON 数据文件
```

## 四、链接管理策略（重点）

### 分类逻辑

```
所有 1835 条微信链接
    ↓ 去重（按 URL 参数 mid+idx+sn）
    ↓ 比对
┌──────────────────┬──────────────────┐
│    已收录（站内）   │   未收录（外部）   │
│  转为站内跳转链接   │  收录到链接库页面   │
│  文内直接可点击     │  标注被引用次数     │
│                    │  支持状态标记       │
│                    │  "待购买/已忽略"    │
└──────────────────┴──────────────────┘
```

### "待收录"链接库页面功能

- 按引用次数排序（被多篇文章引用的链接优先级更高）
- 显示链接标题（从 Markdown 链接文本提取）
- 按来源公众号分组
- 可标记状态：🔖 待购买 / ✅ 已收录 / ❌ 已忽略
- 支持搜索和筛选

## User Review Required

> [!IMPORTANT]
> 以下问题会直接影响开发方向，请给出你的想法：

### 1. 视觉风格选择
**方案 A（墨韵书斋 — 中式文人风）** vs **方案 B（现代阅读器 — 类 Notion 风）**？或者有其他偏好？

### 2. 技术方案确认
是否同意用**纯静态站**方案？还是希望用框架（如 Vite + Vue）以便未来扩展？

### 3. 未来文章增量
后续是否会持续新增文章？如果是：
- 新文章是什么格式导入的？（继续保存成 Markdown？）
- 大约什么频率？（偶尔几篇 / 定期批量导入）

### 4. 评论区的现状
从我看到的几篇来看，评论区都是"暂无评论"。是否有部分文章确实有评论内容？如果有，评论是什么格式？

### 5. 部署方式
- 本地双击打开即可？
- 还是需要部署到 GitHub Pages / Vercel 等在线访问？

### 6. 排版中加粗文字的处理
原文中有大量 `**加粗**` 标记用于强调。你是否希望用颜色高亮来代替或补充加粗？

### 7. 公众号来源分组
文章来自 4 个不同公众号名（记忆承载/记忆承载3/人间罗盘/西风的罗盘），但作者都是碧树西风。需要按公众号分组展示，还是合并为同一作者？
